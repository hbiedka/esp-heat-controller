Heat Controller Object Model

This page provides an overview of the object model used in the Heat Controller project. The object model defines how different components of the system interact and manage their data.

ObjectModelItem Structure

struct ObjectModelItem {
    std::variant value; // The value can be an int, string, or a pointer to another ObjectModel
    ObjectModelSetter* setter; // Pointer to a setter function for this item
    bool saveToNVM; // Indicates whether this item should be saved to non-volatile memory
    size_t nvmStrLen; // If this item is a string that should be saved to NVM, this indicates the maximum length
    bool secret; // Indicates whether this item contains sensitive information (e.g., passwords)
};

ObjectModel Class

class ObjectModel {
public:
    std::map items; // Map of item labels to their corresponding ObjectModelItem
    std::map linkedObjects; // Map of labels to linked ObjectModels (for nested structures)
};

Example Usage

In the Heat Controller project, the ObjectModel is used to represent various components such as WiFi settings, temperature sensors, and control parameters. Each component can have its own ObjectModel with specific items and linked objects.

// Example of creating a WiFi ObjectModel
ObjectModel wifiModel;
wifiModel.items["ssid"] = ObjectModelItem::createStringItem("", &ssidSetter, true, 32);
wifiModel.items["passwd"] = ObjectModelItem::createSecretStringItem("", &passwdSetter, true, 64);

In this example, the WiFi ObjectModel has two items: "ssid" and "passwd". The "ssid" is a regular string item that will be saved to NVM, while the "passwd" is a secret string item that will also be saved to NVM, but will be treated as sensitive information. The setter functions (e.g., &ssidSetter and &passwdSetter) are responsible for updating the values of these items when they are changed.
